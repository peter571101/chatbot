# 妈妈人格 · 实时上下文注入

## 一句话解释

妈妈人格不知道现在几点、什么天气——所以在发消息给 AI 之前，系统会自动查好时间和天气，塞进对话里告诉 AI。

---

## 为什么需要这个机制

LLM 本身没有「现在几点」的概念，也不知道你所在城市的天气。如果不注入这些信息，对话会变成：

> **用户**：妈我出门了
> **AI 妈妈**：好的注意安全

注入上下文后：

> **系统悄悄告诉 AI**：现在是早上 7:30，北京 3°C，大风蓝色预警
> **用户**：妈我出门了
> **AI 妈妈**：哎哟这么早！今天才三度还刮大风，你那件薄外套根本不行，去把羽绒服换上！围巾也戴上！

---

## 消息结构

```
┌─────────────────────────────────────────┐
│ [system] 妈妈人格 system_prompt         │  ← 定义妈妈怎么说话
├─────────────────────────────────────────┤
│ [system] [系统实时信息]                  │  ← 注入的时间+天气（AI 可见）
│ 现在是2026年5月18日 周一 北京时间15:22  │
│ 北京当前天气：晴，气温25°C...           │
├─────────────────────────────────────────┤
│ [user] 妈我晚上不回家吃饭了              │  ← 用户消息
│ [assistant] 又不回来！跟谁吃去啊...      │  ← AI 妈妈的回复
└─────────────────────────────────────────┘
```

核心：上下文是一条 **system 角色的消息**，对用户不可见，但 AI 完全能看到并自然运用。

---

## 触发条件

在 `main.py` 中，当 persona JSON 满足以下任一条件时触发注入：

```python
location = persona.get("location")      # 有城市配置
if persona.get("needs_context") or location:  # 或显式声明需要上下文
    context_text = await build_context(location=location or "北京")
    messages.append({"role": "system", "content": context_text})
```

| JSON 字段 | 类型 | 说明 |
|-----------|------|------|
| `needs_context` | `bool` | 设为 `true` 即开启注入 |
| `location` | `string` | 城市名，如 `"深圳"`、`"上海"`，默认 `"北京"` |

---

## 天气数据来源

使用 [wttr.in](https://wttr.in) 免费天气 API，无需注册或 API Key。

请求格式：
```
GET https://wttr.in/北京?format=j1
```

返回当前天气 + 今天/明天预报，包含：天气描述、气温、体感温度、湿度、风速。

注入给 AI 的文本示例：
```
北京当前天气：晴，气温25°C（体感26°C），湿度45%，风速12km/h。
今天气温范围15°C ~ 28°C。
明天：多云，气温14°C ~ 24°C。
```

缓存策略：同一城市的天气缓存 **10 分钟**，避免频繁请求。

## 如何给其他角色也加上

只需在 persona JSON 中加两行：

```json
{
  "id": "coach",
  "name": "健身教练",
  ...
  "needs_context": true,
  "location": "广州"
}
```

不设置这两个字段的角色不受任何影响。
